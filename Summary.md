# Отчёт по итогам автоматизации

## Что было запланировано и что реализовано
**Запланировано:**
* Поднять тестовое окружение (СУБД MySQL и симулятор банковских сервисов) через Docker.
* Настроить фреймворк для UI-тестирования (Java, Selenide, JUnit 5).
* Реализовать паттерн Page Object для взаимодействия со страницами.
* Подготовить вспомогательные классы (`DataHelper`, `SQLHelper`) для генерации данных и запросов к БД.
* Написать автоматизированные UI-тесты для позитивных и негативных сценариев.
* Подключить генерацию отчетности Allure.
* Настроить CI/CD (GitHub Actions).

**Реализовано:**
Запланированный объем работ выполнен на 100%.

## Причины, по которым что-то не было реализовано
Нереализованных пунктов нет. План выполнен в полном объеме.

## Сработавшие риски
В ходе работы сработали следующие технические риски:
1. **Конфликт версий ПО:** Возник конфликт между установленной на локальной машине версией браузера Chrome (v146) и версией ChromeDriver. Риск нивелирован обновлением библиотеки `Selenide` до актуальной версии `6.19.1`.
2. **Проблемы при компиляции:** Возник конфликт библиотеки Lombok при компиляции проекта (ошибка `JCTree`). Риск устранен путем повышения версии Lombok до `1.18.32` в `build.gradle`.
3. **Сложности с настройкой CI (Linux):** При переносе проекта на сервер GitHub возникли проблемы с правами доступа к скрипту сборки (`Permission denied`), а также с устаревшим синтаксисом Docker. Ошибки устранены добавлением команды выдачи прав `chmod +x gradlew` и обновлением команды запуска на `docker compose`.
